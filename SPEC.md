# 需求规格书

> 一切讨论以本文为准。本文只说「要什么」，不说「怎么实现」。

## 架构

```
微信 ←→ Hermes 默认 profile（Linux 服务器）
               │
               ├── AI 拆解任务 + 安排时间
               │
               ├── 读写日历（Google Calendar API）
               │
               └── 日常监督（定时检查→微信催促）

Windows 11（主力工作机）
  ├── 日历 App（查看 + 弹窗提醒）
  ├── ActivityWatch（活动追踪）
  └── 走神检测脚本（AW 数据 → 判断 → 弹窗/推送）

iOS / Android
  └── 日历 App（仅查看）
```

- **服务器**：只做 AI + 逻辑处理。不存界面、不存数据。
- **Windows**：平时看日历 + 收弹窗提醒。活动追踪在本地跑。
- **手机/iPad**：打开日历 App 就能看到。不操作。
- **微信**：连接默认 profile，用作输入入口和催促通知。不是实时通知频道。

## 用户场景

典型的 ADHD + 阿斯伯格用户。
- 启动困难：知道要做什么，但难以开始
- 需要外部推动：不是「被提醒一下」，是「被追问到动起来」
- 工具不能增加负担：如果维护工具比用它省的时间还多，就是失败

## 使用方式

```
微信发一句话 → AI 拆解 + 安排时间 → 出现在我的日历上 → 到点催我开工 → 没开工就追问 → 走神了提醒
```

用户只跟微信说话，只看日历。中间所有处理对用户透明。

## 需求清单

### R1: 远程任务录入

输入：微信自然语言
输出：日历事件（带时间 + 持续时长）

```
"下周五之前交稿，今天练一小时人体，下午修两个bug"
    → 10:00-11:00 人体练习
    → 14:00-16:00 修登录bug
    → 16:00-17:00 修样式bug
```

不接受：要用户自己拖拽、填时间、分标签。

### R2: AI 拆解

大任务 → 小步骤。每一步都有独立的时间段。

```
"登录功能重做"
    → 阶段1: 分析现有代码（今天 2h）
    → 阶段2: 写新逻辑（明天 3h）
    → 阶段3: 测试（后天 1h）
```

不接受：一个未拆解的标题丢在列表里。

### R3: 到点提醒 + 拖延检测

日历时间到了 → 检查是否开工 → 没开工就追问。

**任务分两种，提醒规则不同：**

| 类型 | 关键词 | 判断逻辑 | 例子 | 提醒规则 |
|---|---|---|---|---|
| 启动任务 | 出门、运动、洗澡、买菜、理发、拿快递等 | 关键词命中 → 强制启动 | 出门 30min | 到时间后 5 分钟还没动 → 微信催 |
| 持续任务 | 画画、编程、修bug、写、练等 | 非启动词 → 按 AI 判定 | 画画 2h | 摸鱼累计 15 分钟 → 提醒，之后每 +5 分钟再提醒 |

- 启动任务：只要开了头就不管了，不需要监督过程
- 持续任务：整个时间段内都在监测走神

追问可以递增：第一次温和 → 第二次直接 → 第三次换方式。

### R5: 走神提醒

正在工作时间，检测到在刷娱乐内容超时 → 提醒。

检测在 Windows 本地跑（ActivityWatch）。检测结果可上报给服务器，由服务器判断 + 推送催促到微信。

不需要阻止，只需要提醒。用户可以选择继续休息。

### R6: 工时记录

自动记录每天在各分类上的时间：
- 绘画（CSP、Procreate）
- 编程（VS Code、终端）
- 摸鱼（娱乐网站、游戏）

不需要用户手动按计时器。

### R7: 跨设备同步

- Windows（主力工作机）上能看日历和任务
- iPad 上能看
- 数据一致，不需要手动同步

### R8: 每日摘要

每天晚上或早上：今天/明天的计划摘要，推送到微信。

### R9: 成就反馈

- 连续完成天数
- 本周累计工时
- 鼓励性质的正面反馈

对 ADHD 用户很重要：需要看到进展，否则容易放弃。

## 不需要的东西

- 精细项目管理（标签体系、优先级矩阵、自定义视图）
- 手动排期（拖拽、填表单）
- 团队协作
- 复杂的统计报表
- 番茄钟
- 多层级项目结构

## 工具原则

- 优先现有开源工具，不从头开发
- 脚本/插件/API 缝合适配，不修改工具源码
- 4GB 服务器，总内存仅剩约 1-1.5GB 可用，不得部署重型服务（Docker 多容器、数据库等）
- 微信是任务输入入口，不需要频繁推送通知到微信
- Windows 11 是主力工作机（活动追踪、日历主界面、提醒弹窗在此）
- iOS / Android 仅查看日历，以防错漏，不做操作
